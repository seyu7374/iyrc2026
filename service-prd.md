# service-prd.md

## 한 줄 정의

**IYRC 대회 운영진이 참가자 정보를 엑셀로 올리고, 심사위원들의 점수를 입력받아 자동으로 순위를 산출하고 출력하는 도구**

---

## Phase 1 — MVP (작동 확인용 최소 범위)

### 핵심 동작

1. **종목 선택**
   - 미리 정의된 종목 목록에서 선택
   - 각 종목별 참가자 정보는 사전에 로드됨

2. **점수 입력**
   - 심사위원이 "심사 모드 ON" 후 참가자별 점수 입력
   - 입력된 점수는 실시간으로 저장됨

3. **자동 순위 산출**
   - 점수 입력 즉시 자동으로 순위 결정
   - 동점(같은 점수)은 같은 순위로 표시

4. **순위표 출력**
   - 최종 순위를 정렬된 표 형태로 화면에 표시
   - 화면 캡처 또는 인쇄로 공유 가능

5. **AI 점수 검증** (선택적)
   - 심사위원이 버튼 클릭 시 AI가 이상값 탐지
   - 예: 다른 참가자의 평균 점수와 비교해 이상하게 높거나 낮은 점수 경고
   - 심사위원이 확인 후 수정할 수 있도록 제안

### 사용자 흐름

```
심사위원 → 종목 선택 → 심사 모드 ON → 점수 입력
        → [AI 검증 버튼 (선택)] → 자동 순위 산출 → 출력/캡처
```

### 성공 기준

✅ **"종목을 선택한 후 점수를 입력하면 순위가 자동으로 나오고 출력할 수 있으면 성공"**

---

## Phase 2 — 참가자 정보 관리 (엑셀 업로드) + 여러 심사위원

### 추가 기능

1. **엑셀 파일 업로드**
   - 운영진이 종목별 참가자 명단을 엑셀로 업로드
   - 자동으로 각 종목의 참가자 목록이 갱신됨

2. **여러 심사위원 지원**
   - 같은 종목에 여러 심사위원(1~3명)이 각각 점수 입력 가능
   - UI: 심사위원 구분 탭 또는 이름별 입력 칼럼
   - 여러 심사위원의 점수를 자동으로 합산 후 평균 계산
   - 평균 점수로 최종 순위 결정

### Excel 파일 포맷 (필수)

운영진이 업로드하는 엑셀 파일은 다음 구조를 따름:

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| A | No. | 참가자 번호 | 1004, 1201, 1409 |
| B | Country | 국가 | Korea, Algeria, India, Israel |
| C | Passport name | 영문 이름 | John Smith, NISHAL SAI KRISHNACHAITANYA |
| D | Date of Birth | 생년월일 | 2010-05-15, 2014-02-22 |
| E | Gender | 성별 | M, F |
| F | 종목명/School | 학교 또는 종목 | Robot Soccer, X, YES |
| G | Game No. | 게임/조 번호 | A, B, C, RS-01, RS-02, RS-03, YES, X |

**팀 구성 규칙 (Game No. 기반):**
- **개인전**: Game No. = "X" 또는 공백 → 모두 teamNo = 1
- **팀전**: Game No. = "A"/"B"/"C"/"D" 또는 "RS-01"/"RS-02"/"RS-03" → 해당 팀으로 자동 그룹화
- **YES**: Game No. = "YES" → teamNo = 1 (기본 팀)

**주의사항:**
- 첫 번째 행은 헤더
- 두 번째 행부터 데이터 입력
- 셀 병합 가능 (자동 처리됨)
- .xlsx 또는 .xls 형식 지원

### 성공 기준

✅ **"엑셀 파일을 업로드하면 참가자 정보가 정확히 로드되고, 여러 심사위원의 평균 점수로 순위가 결정되어야 함"**

## Phase 3 — 권한 관리 및 사용자 구분

### 추가 기능

- **로그인 시스템**
  - 심사위원/운영진: 이메일 + 비밀번호 로그인 가능
  - 점수 입력 권한 가짐

- **사용자 권한 구분**
  - 심사위원: 점수 입력, 수정 가능
  - 운영진: 참가자 관리, 심사위원 추가, 순위 인쇄
  - 관중/사용자: 읽기 전용 (공개 링크로 실시간 순위만 확인)

- **감시 기능**
  - 운영진이 심사위원별 점수 입력 현황 모니터링
  - 입력 이력 기록 (누가 언제 어떤 점수를 입력했는지)

### 성공 기준

✅ **"심사위원 로그인 후 점수 입력, 운영진이 모니터링, 관중은 공개 링크로 순위만 볼 수 있어야 함"**

---

## Phase 4 — 데이터 지속성 및 협업 강화

### 추가 기능

- **Supabase 데이터베이스**
  - 현재 로컬 상태 → 클라우드 DB 전환
  - 여러 심사위원이 동시에 점수 입력 가능 (실시간 동기화)
  - 변경 이력 자동 저장 (누가 언제 무엇을 변경했는지)

- **대회 관리**
  - 여러 대회를 동시에 관리 가능
  - 대회별 설정 (심사위원 수, 점수 범위, 점수 반올림 방식 등)

- **알림 기능**
  - 심사위원에게 "점수 입력 요청" 알림
  - 점수 입력 완료 시 운영진에게 알림

### 성공 기준

✅ **"여러 심사위원이 동시에 점수를 입력해도 중복 없이 정확한 최종 순위가 나와야 함"**

---

## Phase 5 — AI 고도화 및 자동 분석

### 추가 기능

- **AI 자동 평가 의견**
  - 심사위원 점수에 기반해 Gemini API로 각 참가자 평가 의견 자동 생성
  - 예: "이 팀은 창의성이 뛰어났지만 기술 구현에서 개선 필요"

- **순위 분석 리포트**
  - AI가 최종 순위 결과를 분석해 자동 리포트 작성
  - "심사위원 간 점수 편차가 큰 팀", "만장일치로 선정된 팀" 등 인사이트

- **점수 정규화**
  - 심사위원별 점수 편향성 자동 감지 및 보정
  - 예: 한 심사위원이 자주 높은 점수를 주는 경향 보정

- **제안 시스템**
  - AI가 "이 점수는 이상하지 않을까?" 하고 심사위원에게 재검토 제안

### 성공 기준

✅ **"AI가 제안한 재검토 사항이 실제 채점 오류를 95% 이상 감지해야 함"**

---

## 기술 스택 (고정)

- **프론트엔드**: Next.js 16 (App Router) + TypeScript + Tailwind CSS
- **백엔드**: Supabase (Phase 4부터)
- **AI**: Gemini API
- **배포**: Vercel
- **폰트**: Pretendard (한국어 최적화)

---

## 주요 고려사항

1. **오프라인 대비**: 네트워크 끊김 시에도 로컬에서 점수 입력 후 나중에 동기화
2. **빠른 응답**: 점수 입력 시 1초 이내 순위 갱신
3. **정확성**: 점수 계산 오류 방지 (자동 검증)
4. **공정성**: 모든 심사 과정이 기록되고 투명하게 추적 가능
5. **사용 편의성**: 심사위원/운영진 대상이므로 복잡하지 않게 설계

---

**최종 목표**: Phase 1을 완성하면 **기본 점수 관리 도구로 작동**, Phase 2~5를 순차적으로 얹으면 **대규모 국제 대회까지 대응 가능한 종합 시스템**이 되어야 함.
