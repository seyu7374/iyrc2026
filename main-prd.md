# 메인페이지 기획서 (main-prd.md)

## 📋 개요

**페이지명**: 로봇 대회 점수 관리 - 메인페이지 (`/`)  
**목적**: 대회 운영진에게 자동 점수 집계·순위 산출 기능의 가치를 전달하고 무료 가입으로 유도  
**타겟**: IYRC 등 로봇 대회 운영진, 대회 조직위원회, 학교 행사 담당자  
**디자인 톤**: 모던 다크 모드 (신뢰성 + 전문성 + 기술성)

---

## 🎯 핵심 메시지

### 주제
심사 점수를 **수동으로 집계하는 대신, 자동으로 정리되게 하세요**

### 서브 메시지
- "심사위원이 입력하면 자동 집계"
- "실시간 순위 산출"
- "계산 오류 제거"
- "대회 운영 시간 단축"

- 구성: 로고(좌) + 네비게이션(중) + CTA 버튼(우)


**세부사항**:
- 배경: 검은색
- 5개 항목 (또는 4개)

- 레이아웃: 세로 리스트 또는 그리드

**항목 상세**:
| # | 문제 | 설명 |
|---|------|------|
| 1 | 📝 수작업 | 심사위원이 제출한 점수를 엑셀에 일일이 입력 |

| 3 | ⚠️ 오류 가능성 | 계산 실수, 중복 입력, 형식 오류로 재검산 필요 |
| 4 | ⏰ 시간 낭비 | 모든 과정이 오래 걸려 최종 결과 발표 지연 |
| 5 | 😰 신뢰성 문제 | 수동 작업이라 공정성 논란 가능성 |

### 4️⃣ 솔루션 섹션 (Solution/How It Works)
```
┌────────────────────────────────────┐
│  자동으로 정리되는 프로세스         │
│                                    │
│  심사위원 입력 → 자동 집계 → 순위산출
│     (클라우드)   (실시간)   (자동)
│                                    │
│  ✓ 오류 제거                      │

│  ✓ 공정성 보장                    │
│                                    │
└────────────────────────────────────┘
```

**세부사항**:
- 프로세스 흐름: 3단계 (입력 → 집계 → 순위)
- 각 단계 아래 특징 표시
- 아래 결과: 3개 체크 항목

**복사**:
- 흐름: "심사위원이 온라인에서 점수 입력 → 자동으로 집계 → 순위 즉시 산출"
- 특징:
  1. ✓ **오류 제거**: 계산 실수 제거, 자동 검증
  2. ✓ **시간 단축**: 몇 초 만에 최종 순위 확정
  3. ✓ **공정성 보장**: 모든 과정이 시스템에 기록, 투명성 확보

## 🎨 디자인 가이드

### 색상 팔레트
| 용도 | 색상 | 값 |
|------|------|-----|
| 배경 | 검은색 | #000000 |
| 배경 (보조) | 어두운 회색 | #111111 / #1a1a1a |
| 텍스트 (주) | 흰색 | #FFFFFF |
| 텍스트 (보조) | 밝은 회색 | #999999 / #CCCCCC |
| 테두리 | 진회색 | #333333 |
| 강조 | 흰색 (버튼) | #FFFFFF |

### 폰트
- 모든 텍스트: **Pretendard** (한국어 최적화)
- 헤딩: 700 (Bold)
- 본문: 400 (Regular) / 500 (Medium)

### 버튼 스타일
| 타입 | 배경 | 텍스트 | 호버 | 용도 |
|------|------|--------|------|------|
| Primary | 흰색 | 검은색 | 밝은 회색 | 가입, 시작 |
| Secondary | 투명 | 흰색 | 회색 테두리 | 데모, 학습 |

### 간격 및 레이아웃
- 섹션 간 상하 여백: 80-120px
- 내부 패딩: 40-60px (좌우), 60-80px (상하)
- 카드 간 간격: 24-32px
- 최대 콘텐츠 너비: 1280px

### 효과
- 호버: 테두리 색상 변경, 배경 밝아짐 (부드러운 전환)
- 아이콘 호버: 스케일 확대 (선택)
- 없음: 과도한 애니메이션
